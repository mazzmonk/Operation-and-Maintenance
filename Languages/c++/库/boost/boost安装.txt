oogle-perftools安装（非必须）
安装
输入命令：
./configure 可安装全部google-perftools的组件，包括TCMalloc、heap-checker、heap-profiler和cpu-profiler共4个组件。
但是我们这里只需要TCMalloc所以我们可以输入命令：
./configure --prefix=/usr/local --disable-cpu-profiler --disable-heap-profiler --disable-heap-checker --disable-debugalloc --enable-minimal 只安装TCMalloc。
--prefix=/usr/local是指定安装目录的，如果不指定则默认安装到/usr/local/lib/下面。


boost-1.54需要glibc-2.14以上的库
glibc-2.18的版本过高，影响整个系统的命令和程序

tar zxvf glibc-2.14.tar.gz
cd glibc-2.14
mkdir build
cd build
../configure --prefix=/usr/local/glibc-2.14
make -j4   
make install  

编译过程出现如下错误：
checking LD_LIBRARY_PATH variable... contains current directory

LD_LIBRARY_PATH不能以终结符作为开始和最后一个字符，不能有2个终结符连在一起，我的LD_LIBRARY_PATH为 :/usr/local/firefox:/usr/local/firefox,只要在前面加上一个路径，不让：出现在第一个字符就可以了 ”
查看环境变量，真会出现上述错误，因此修改如下
export LD_LIBRARY_PATH=/usr/local/mpc-1.0.1/lib:/usr/local/gmp-5.1.2/lib:/usr/local/mpfr-3.1.2/lib

装完以后设置如下

export LD_LIBRARY_PATH=/usr/local/mpc-1.0.1/lib:/usr/local/gmp-5.1.2/lib:/usr/local/mpfr-3.1.2/lib:/usr/local/glibc-2.14/lib
