configure: error: Package requirements (libffi >= 3.0.0) were not met:

Consider adjusting the PKG_CONFIG_PATH environment variable if you

Alternatively, you may set the environment variables LIBFFI_CFLAGS

解决：
cp /usr/local/libffi/lib/pkgconfig/libffi.pc /usr/lib/pkgconfig/
cp /usr/local/libffi/lib/pkgconfig/libffi.pc /usr/lib64/pkgconfig/


指定libiconv安装路径
./configure --with-libiconv=/usr/local/libiconv时提示如下错误
No iconv() implementation found in C library or libiconv

解决：

./configure --with-libiconv=gnu CPPFLAGS="-I/usr/local/libiconv/include" LDFLAGS="-L/usr/local/libiconv/lib"


关于pkg-config的解决

安装好相关软件以后，在安装好的软件目录中有pkg-config目录，将里边的东西全部拷贝到/usr/lib/pkgconfig及/usr/lib64/pkgconfig目录下
然后执行命令pkgconfig xxx 






./configure --prefix=/usr/local/librsvg-2.37.0  LIBRSVG_CFLAGS="/usr/lib/" LIBRSVG_LIBS="/usr/lib" --disable-Bsymbolic
